# model_comparison_current

Model-map speckle (neighbor-change) on the current pipeline, replacing the speckle portion of the
earlier `reports/model_comparison/` arm (which used the static 2018-2020 mosaic on a 154-location
snapshot with random dedup). Generated by `scripts/model_speckle_current.py`.

## Basis

- Classified rasters: the current temporally-matched per-bracket predictions,
  `data/raw/transfer_predictions/<bracket>/pred_<bracket>_cell<id>.tif`, bands 1 to 5 = v2 to v6, over
  the 180 adjudicated cells (adjudicated reference, correct embedding bracket per cell).
- neighbor-change is the fraction of horizontally-adjacent, both-valid pixel pairs whose class differs,
  pooled over all 180 cells. Coverage is 1.0, since the per-bracket cells are fully classified, unlike
  the earlier static mosaic (coverage about 0.69).

## Results

| version | neighbor-change | pooled OA | earlier snapshot |
|---------|-----------------|-----------|------------------|
| v2 | 0.085 | 0.659 | 0.074 |
| v3 | 0.084 | 0.635 | 0.077 |
| v4 | 0.155 | 0.245 | 0.135 |
| v5 | 0.093 | 0.594 | 0.091 |
| v6 | 0.801 | 0.130 | 0.781 |

Current values run modestly higher than the earlier snapshot but preserve the ordering and the smooth
versus speckly character. Pooled OA is the current pooled overall accuracy against the adjudicated
reference (from comparison/overall_comparison.csv), so the table and figure show the speckle versus
accuracy relationship: the smooth variants (v2, v3, v5) sit at low speckle and higher accuracy, and
the speckly v6 sits at high speckle and low accuracy.

## What was and was not reproduced

- Reproduced on the current basis: the neighbor-change speckle table (feeds Table S3) and the
  speckle-versus-accuracy figure.
- Not reproduced: the per-version accuracy and confusion matrices, which already exist on the current
  adjudicated per-bracket basis in `reports/transfer_confusion_adjudicated/` and
  `comparison/overall_comparison.csv` (Tables 2.3 and 2.4). The dedup-selection sensitivity is moot
  under adjudication, since each cell has exactly one chosen reviewer, so it was not recomputed.

## Outputs

- `model_speckle.csv`: per variant, neighbor_change, valid_pairs, differing_pairs, total_pairs,
  coverage, and pooled_OA.
- `speckle_vs_accuracy.png`: neighbor-change against pooled overall accuracy per variant.
