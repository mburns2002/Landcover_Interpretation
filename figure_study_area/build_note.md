# Figure 1 build note

Generated by `scripts/build_study_area_figure.py`.

## Input layers found

All three requested layers were located and used, none is missing.

- GLKN watershed boundaries: `GLKN_watershed_boundaries_7park_5070.shp`, downloaded from Google Drive
  `GLKN_grid_analysis/` to `data/raw/glkn/`. Already in EPSG:5070, 7 features, with a `park` attribute
  taking the values apis, grpo, isro, miss, sacn, slbe, and voya.
- Study grid: `grid_112_naip_brackets_5_11_26.csv`, downloaded from the Google Drive root to
  `data/raw/glkn/`. 21,561 rows, one per grid cell, with an integer `id` and EPSG:5070 cell bounds in
  the `left`, `right`, `top`, and `bottom` columns, from which each cell polygon is built directly.
  Similarly named files exist in Drive (`grid_112_naip_brackets_clean_5_11_26.csv`,
  `grid_112_naip_brackets.csv`, and copies under `GLKN_grid_analysis/` and `Spring_2026/`); only the
  exact file named in the request was used, and none were merged.
- Interpreted cells: `exports/gee/interpreted_cells_by_bracket.csv`, already local. 180 rows, with
  `cell_id` (zero-padded 5-digit string) and `bracket`.

## Interpreted-cell join (verified)

The interpreted cells were joined to the grid polygons on the cell key. The grid `id` was cast to a
zero-padded 5-digit string (`str(id).zfill(5)`) to match the `cell_id` padding exactly. The join
returned **180 of 180** cells, matching the expected count, so the cell layer is complete and no cell
was silently dropped. The build stops with an error if the join count differs from the expected
count.

## Basemap and layers

- Basemap: Natural Earth vector layers, not raster tiles, since this is line and vector cartography
  and a vector basemap keeps the figure fully vector. Layers used: `ne_50m_admin_1_states_provinces_
  lakes` (state and province context), `ne_50m_lakes` (Great Lakes), and `ne_110m_admin_0_countries`
  (locator inset), downloaded to `data/raw/naturalearth/`. Contextily tiles were available but not
  used.
- Layer order, back to front: states and provinces, Great Lakes, study grid extent (dissolved to one
  polygon), the seven GLKN park boundaries (distinct color per park), and the 180 interpreted cells.

## Projection and cartographic elements

- Projection: EPSG:5070 (CONUS Albers, equal-area). Every layer is reprojected to 5070 before
  plotting.
- Scale bar: units are kilometers (5070 units are meters, auto-labeled in kilometers at this range).
- North arrow: included, placed over open water.
- Locator inset: conterminous United States with a red box marking the main-map extent.

## Figure dimensions and format

- Size: 7.50 x 6.70 inches (double-column width; no RSE style module exists in the repository, so a
  standard double-column width was used). The bottom strip holds the legend.
- Raster: `figure1_study_area.png` at 300 dpi.
- Vector: `figure1_study_area.pdf`.

## Flag for the manuscript introduction (not part of the caption)

This figure shows seven GLKN park units. The full NPS GLKN roster is nine units; INDU (Indiana Dunes)
and PIRO (Pictured Rocks) are excluded here for falling outside the study grid extent. The manuscript
introduction should state the GLKN unit count consistently with this figure. If the introduction says
nine without noting the two exclusions, the figure and the text will disagree, so reconcile the count
or add the exclusion note in the introduction.
